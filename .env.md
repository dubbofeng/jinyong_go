# 环境配置说明

## 环境变量配置

### 本地开发 (.env.local)

将以下环境变量添加到 `.env.local` 文件：

```env
# PostgreSQL 数据库配置（本地开发）
POSTGRES_URL=postgresql://postgres:postgres@localhost:5432/jinyong_go
POSTGRES_PRISMA_URL=
POSTGRES_URL_NON_POOLING=
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/jinyong_go

# NextAuth Secret (生成: https://generate-secret.vercel.app/32)
AUTH_SECRET=

# Google Gemini API (获取: https://console.cloud.google.com/)
GEMINI_API_KEY=your-gemini-api-key
```

## 数据库设置

### 1. 本地 PostgreSQL 安装

```bash
# macOS (使用 Homebrew)
brew install postgresql@15
brew services start postgresql@15

# 创建数据库
createdb jinyong_go

# 验证连接
psql postgresql://postgres:postgres@localhost:5432/jinyong_go
```

### 2. 数据库迁移

```bash
# 生成迁移文件
pnpm db:generate

# 执行迁移
pnpm db:push

# 查看数据库（Drizzle Studio）
pnpm db:studio
```

### 3. 数据库操作

```bash
# 检查迁移状态
pnpm db:check-migrations

# 修复序列
pnpm db:fix-sequences

# 播种数据
pnpm db:seed
```

## 开发环境

### 启动开发服务器

```bash
# 安装依赖
pnpm install

# 启动开发服务器（运行在端口 9999）
pnpm dev
```

访问 http://localhost:9999

### 开发脚本

```bash
# 代码检查和格式化
pnpm check          # 执行 lint、格式检查和类型检查
pnpm lint           # ESLint 检查
pnpm lint:fix       # 修复 lint 问题
pnpm format         # 格式化代码
pnpm type-check     # TypeScript 类型检查

# 数据初始化
pnpm init:npcs      # 初始化所有 NPC
pnpm init:maps      # 初始化地图
pnpm generate:maps  # 生成地图并添加 NPC

# 测试
pnpm e2e            # 运行 E2E 测试
pnpm e2e:ui         # 运行 E2E 测试（UI 模式）
```

## 生产环境部署

### Vercel 部署

1. **推送代码到 GitHub**

   ```bash
   git push origin main
   ```

2. **在 Vercel 控制台配置**
   - 导入项目
   - 配置环境变量：
     - `POSTGRES_URL` - Vercel Postgres URL
     - `POSTGRES_PRISMA_URL` - Prisma 连接 URL
     - `POSTGRES_URL_NON_POOLING` - 非连接池 URL
     - `AUTH_SECRET` - NextAuth 密钥
     - `GEMINI_API_KEY` - Google Gemini API 密钥

### 构建和启动

```bash
# 生产构建
pnpm build

# 启动生产服务器
pnpm start
```
