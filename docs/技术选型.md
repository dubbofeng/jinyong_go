## 二、游戏策划文档（简化版）

### 核心设计理念
**轻量级、易部署、低成本**

### 1. 游戏模式调整
- **纯单机模式**：大部分内容本地运行
- **异步社交**：棋谱分享、排行榜等异步功能
- **无需实时对战**：简化网络需求

### 2. 技能系统简化
保留核心8个技能：
1. **亢龙有悔** - 悔棋（基础）
2. **凌波微步** - 试下（基础）
3. **棋魂召唤** - AI提示（使用浏览器端KataGo）
4. **吸星大法** - 移除对手棋子
5. **双手互搏** - 连走两步
6. **乾坤大挪移** - 交换棋子
7. **独孤九剑** - 高亮坏棋（被动）
8. **一阳指** - 死活计算

### 3. 数据存储策略
- **本地存储**：游戏进度、技能状态
- **云端同步**：用户数据、棋谱、排行榜
- **棋谱分析**：浏览器端KataGo计算，结果保存到云端

### 4. 部署优势
- **零服务器成本**：Vercel免费额度
- **无运维负担**：Serverless自动扩展
- **全球访问**：Vercel全球CDN
- **开发简单**：统一技术栈（TypeScript）

### 数据流设计

#### 游戏数据流
```
1. 本地游戏循环
   RPG-JS运行 → 本地状态更新 → Canvas渲染

2. 技能触发
   玩家使用技能 → 围棋模块处理 → 技能效果应用

3. AI计算（浏览器端）
   请求AI分析 → WebAssembly KataGo计算 → 返回建议

4. 数据保存
   游戏关键节点 → API调用 → 保存到PostgreSQL

5. 社交功能
   棋谱分享 → 生成SGF文件 → 上传到Blob Storage
```

// 用户表
export const users = pgTable('users', {
  id: uuid('id').primaryKey().defaultRandom(),
  email: varchar('email', { length: 255 }).unique().notNull(),
  username: varchar('username', { length: 50 }).unique().notNull(),
  createdAt: timestamp('created_at').defaultNow(),

export const gameProgress = pgTable('game_progress', {
  id: uuid('id').primaryKey().defaultRandom(),
  userId: uuid('user_id').references(() => users.id),
  chapter: integer('chapter').default(1), // 当前章节
  location: varchar('location', { length: 50 }), // 当前位置
  unlockedSkills: json('unlocked_skills').$type<string[]>(),
  completedQuests: json('completed_quests').$type<string[]>(),

});

// 棋局记录表
export const chessRecords = pgTable('chess_records', {
  id: uuid('id').primaryKey().defaultRandom(),
  userId: uuid('user_id').references(() => users.id),
  gameType: varchar('game_type', { length: 20 }), // 'story', 'practice', 'challenge'
  boardSize: integer('board_size'), // 9, 13, 19
  moves: json('moves').$type<Array<[number, number, number]>>(), // [x, y, player]
  skillsUsed: json('skills_used').$type<Array<{skill: string, move: number}>>(),
  result: varchar('result', { length: 10 }), // 'win', 'lose', 'draw'
  duration: integer('duration'), // 游戏时长(秒)
  createdAt: timestamp('created_at').defaultNow(),
});

// 技能熟练度表
export const skillMastery = pgTable('skill_mastery', {
  id: uuid('id').primaryKey().defaultRandom(),
  userId: uuid('user_id').references(() => users.id),
  skillId: varchar('skill_id', { length: 50 }),
  level: integer('level').default(1),
  uses: integer('uses').default(0), // 使用次数
  lastUsed: timestamp('last_used'),
});

// 排行榜表
export const leaderboard = pgTable('leaderboard', {
  id: uuid('id').primaryKey().defaultRandom(),
  userId: uuid('user_id').references(() => users.id),
  category: varchar('category', { length: 50 }), // 'wins', 'rating', 'skill_mastery'
  score: decimal('score', { precision: 10, scale: 2 }),
  rank: integer('rank'),
  updatedAt: timestamp('updated_at').defaultNow(),

### KataGo浏览器端集成

#### 技术方案
1. **使用 y-ich/KataGo 的WebAssembly版本**
2. **轻量模型选择**：
   - 9路棋盘：使用最小模型（~10MB）
   - 13路棋盘：中等模型（~30MB）
   - 19路棋盘：标准模型（~100MB，可选加载）

3. **计算策略**：
   - 思考时间限制：3-10秒
   - 计算线程：Web Worker后台计算
   - 缓存机制：常见局面结果缓存

4. **性能优化**：
   - 懒加载模型文件
   - 增量计算（开局库）
   - 计算中断机制（用户操作时停止计算）

### 风险评估与应对

#### 技术风险
1. **KataGo浏览器性能**
   - 风险：低端设备卡顿
   - 应对：提供性能设置，可关闭AI功能

2. **浏览器兼容性**
   - 风险：旧浏览器不支持WebAssembly
   - 应对：检测并提示用户升级浏览器

3. **数据丢失风险**
   - 风险：本地存储被清除
   - 应对：重要数据云端备份，恢复机制

4. **免费额度超限**
   - 风险：用户增长过快
   - 应对：监控使用量，提前升级计划

#### 业务风险
1. **用户留存**
   - 风险：游戏内容消耗过快
   - 应对：持续更新内容，社区活动

2. **商业化困难**
   - 风险：难以盈利
   - 应对：先建立用户基础，再考虑付费功能
