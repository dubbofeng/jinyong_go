### 技术难点突破记录

#### 1. TensorFlow.js在Worker中加载 ✅
**问题**：Worker线程无法访问TensorFlow.js全局变量  
**解决**：
- 使用`importScripts()`在Worker启动时同步加载TF.js
- 检查`ENVIRONMENT_IS_PTHREAD`而非`Module["ENVIRONMENT_IS_PTHREAD"]`
- 添加`typeof tf === 'undefined'`避免重复加载

#### 2. SharedArrayBuffer支持 ✅
**问题**：pthread需要SharedArrayBuffer，需要特殊headers  
**解决**：
- 配置Next.js headers：COOP: same-origin, COEP: credentialless
- credentialless模式允许加载无CORP headers的资源
- 为/katago路径单独配置CORP: cross-origin

#### 3. Worker内存访问 ✅
**问题**：`Module.HEAPF32`在pthread Worker中未定义  
**解决**：
- Worker中使用全局`HEAPF32`而非`Module.HEAPF32`
- 在predict方法中动态检测可用的HEAP引用
- 统一使用`heapF32`变量访问内存

#### 4. GTP协议通信 ✅
**问题**：stdin/stdout重定向和命令响应管理  
**解决**：
- 使用FS.init()重定向stdin/stdout
- 实现Input/Output类管理缓冲区
- 使用Promise + 超时机制处理命令响应
- 将input/output实例赋值给Module全局变量

##### 支持的条件类型
| 条件类型 | 说明 | 示例 |
|---------|------|------|
| level | 等级要求 | `Requirements.level(10, 20)` |
| chapter | 章节要求 | `Requirements.chapter(2)` |
| quest_completed | 任务完成 | `Requirements.questCompleted('quest_id')` |
| npc_defeated | NPC已击败 | `Requirements.npcDefeated('guojing')` |
| skill_unlocked | 技能已解锁 | `Requirements.skillUnlocked('skill_id')` |
| first_time | 首次交互 | `Requirements.firstTime()` |
| affection_level | 好感度等级 | `Requirements.affectionLevel('friend')` |
| and/or/not | 逻辑组合 | `Requirements.and(...)` |
| ... | 共15种 | 详见使用指南 |

##### 配置示例
```typescript
// 在scripts/test-npc-requirements.ts或数据库中配置
const npcRequirements = {
  dialogues: {
    first_meet: {
      unlockConditions: [Requirements.firstTime()],
    },
    learn_skill: {
      unlockConditions: [
        Requirements.and(
          Requirements.level(10),
          Requirements.chapter(2)
        ),
      ],
      lockedHint: '需要10级且达到第二章',
    },
  },
  battle: {
    unlockConditions: [Requirements.level(15)],
    repeatable: true,
    cooldownSeconds: 3600,
  },
};
```
