## 清理前后对比

### 之前的工作流程（已废弃）
```bash
# 1. 从JSON加载数据
pnpm seed

# 2. 初始化地图
pnpm seed:maps

# 3. 完整重置
pnpm db:reset  # 执行 seed + seed:maps
```

**问题**：
- JSON文件数据不完整（只有3个旧NPC）
- 需要手动维护JSON文件
- 数据分散在JSON和数据库两处
- 容易造成数据不一致

### 现在的工作流程（新系统）
```bash
# 1. 初始化NPC（20个，从Post-MVP计划生成）
pnpm init:npcs

# 2. 初始化地图结构
pnpm init:maps

# 3. 生成完整地图内容（NPC自动从数据库读取）
pnpm generate:maps

# 4. 完整重置
pnpm db:reset  # 执行 db:push + init:npcs + init:maps
```

**优势**：
- ✅ 所有数据存储在数据库
- ✅ 单一数据源，无需JSON文件
- ✅ 从Post-MVP计划自动生成
- ✅ 数据完整（20个NPC）
- ✅ 易于扩展和维护

## 新系统架构

### 数据初始化脚本

| 脚本 | 功能 | 输出 |
|------|------|------|
| `init-all-npcs.ts` | 生成20个NPC | npcs表 + items表 |
| `init-complete-maps.ts` | 初始化地图结构 | maps表 |
| `generate-complete-maps.ts` | 生成地图内容 | map_tiles表 + map_items表 |
| `update-world-map-portals.ts` | 更新世界地图传送门 | map_items表 |

### 数据流

```
Post-MVP计划.md
    ↓
init-all-npcs.ts
    ↓
┌───────────────┐
│ npcs表        │ (20个NPC)
│ items表       │ (npc_xxx items)
└───────────────┘
    ↓
generate-complete-maps.ts (从数据库读取NPC)
    ↓
┌───────────────┐
│ map_items表   │ (NPC实例)
└───────────────┘
    ↓
IsometricGame.tsx (从itemId提取npcId)
```

### 对话文件管理

对话文件仍然使用JSON格式（便于翻译和版本控制）：
```
public/dialogues/
  ├── zh/
  │   ├── hong_qigong.json
  │   ├── linghu_chong.json
  │   └── ...
  └── en/
      ├── hong_qigong.json
      ├── linghu_chong.json
      └── ...
```

## 如何使用新系统

### 首次设置
```bash
# 1. 推送数据库Schema
pnpm db:push

# 2. 初始化NPC数据
pnpm init:npcs

# 3. 初始化地图
pnpm init:maps

# 4. 生成地图内容
pnpm generate:maps
```

### 日常开发
```bash
# 添加新NPC
# 1. 编辑 scripts/init-all-npcs.ts 中的 npcData 数组
# 2. 运行: pnpm init:npcs
# 3. 生成图片（AI工具）
# 4. 创建对话文件
# 5. 更新地图: pnpm generate:maps

# 重置所有数据
pnpm db:reset
```
