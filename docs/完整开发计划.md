# 完整开发计划文档

## 项目总体规划

**项目名称**：《金庸棋侠传》
**总体周期**：12-16周（三个版本迭代）
**技术栈**：Next.js 14 + RPG-JS v4 + KataGo WASM + PostgreSQL + Vercel
**目标用户**：围棋爱好者、武侠迷、策略游戏玩家

---

## 版本规划

### V1.0 - MVP（4-6周）
**核心目标**：验证游戏概念和核心玩法

**包含内容**：
- 5个基础技能
- 第一章内容（华山、少林寺、襄阳城）
- 3个主要NPC
- 9路围棋对战
- 6个主线任务
- 基础等级系统（1-20级）
- 单机游戏模式

**详见**：MVP开发计划.md

### V1.5 - 功能完善（4-6周）
**核心目标**：扩展游戏深度，引入社交功能

**主要功能**：
- 第二、三章内容解锁
- 更多NPC和地点（5+新NPC）
- 新增4-5个战术技能
- 13路棋盘支持
- 异步PVP对战系统
- 棋谱分享功能
- 排行榜系统
- 好友系统
- 支线任务系统

### V2.0 - 完整体验（4-6周）
**核心目标**：完成全部内容，打磨游戏体验

**主要功能**：
- 第四、五章完成
- 全部13个技能实装
- 19路棋盘支持
- 完整排行榜系统
- 棋社系统
- 师徒系统
- 内力系统（技能消耗资源）
- 外观系统（棋盘皮肤、棋子样式）
- 高级任务（死活题、官子题专项）
- 移动端优化（PWA）

## Isometric 地图系统后续开发建议

### 高优先级（立即执行）

#### 1. 玩家移动系统（预计3-4天）
**目标**：让玩家能在 Isometric 地图上自由移动

**实现步骤**：
1. **点击移动**
   - 监听地图瓦片点击事件
   - 计算点击位置的网格坐标
   - 触发角色移动到目标位置

2. **路径寻找**
   - 实现 A* 算法或 Dijkstra 算法
   - 考虑地形可行走性（water 不可走）
   - 避开建筑和障碍物

3. **移动动画**
   - 平滑插值移动（lerp）
   - 角色朝向调整
   - 行走动画播放
   - 移动速度配置（建议：200ms/格）

4. **碰撞检测**
   - 检查目标瓦片类型
   - 检查是否有建筑物阻挡
   - 检查 NPC 位置
   - 边界检测

**技术要点**：
```typescript
// 伪代码示例
function handleTileClick(gridX: number, gridY: number) {
  // 1. 检查目标是否可行走
  if (!isWalkable(gridX, gridY)) return;
  
  // 2. 计算路径
  const path = findPath(player.x, player.y, gridX, gridY);
  
  // 3. 执行移动动画
  animateMovement(path);
}
```
