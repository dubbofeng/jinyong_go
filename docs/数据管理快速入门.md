# 数据管理快速入门

## 🚀 快速开始

### 1. 首次设置（新环境）

```bash
# 1. 推送数据库结构
pnpm db:push

# 2. 加载所有种子数据（NPC + 任务 + 地图）
pnpm seed:all

# 3. 启动开发服务器
pnpm dev
```

### 2. 常用命令

```bash
# 加载 NPC 和任务数据
pnpm seed

# 加载示例地图数据
pnpm seed:maps

# 加载所有种子数据
pnpm seed:all

# 重置数据库到初始状态
pnpm db:reset

# 打开数据库可视化界面
pnpm db:studio
```

## 📝 修改数据

### 修改对话内容（最常用）

```bash
# 1. 编辑对话文件
vim src/data/dialogues/hong_qigong.zh.json

# 2. 刷新浏览器即可看到效果
# （开发模式下自动热更新，无需重启）
```

### 修改 NPC 配置

**方式一：修改 JSON 文件**
```bash
# 1. 编辑 NPC 配置
vim src/data/npcs.json

# 2. 重新加载种子数据
pnpm seed

# 3. 刷新浏览器
```

**方式二：通过后台管理（推荐）**
```bash
# 1. 访问后台管理界面
open http://localhost:9999/admin

# 2. 在界面上直接修改 NPC 数据
# （未来功能，正在开发中）
```

### 修改任务配置

```bash
# 1. 编辑任务文件
vim src/data/quests.json

# 2. 重新加载
pnpm seed

# 3. 刷新浏览器
```

### 创建新地图

```bash
# 方式一：通过后台地图生成器（推荐）
open http://localhost:9999/admin/maps

# 方式二：通过脚本创建
# 编辑 scripts/init-maps.ts，添加新地图配置
pnpm seed:maps
```

## 🐛 故障排除

### 问题：NPC 不显示

```bash
# 1. 检查数据是否加载
pnpm db:studio
# 查看 npcs 表是否有数据

# 2. 重新加载种子数据
pnpm seed

# 3. 检查 NPC 位置配置
cat src/data/npcs.json | grep -A 5 "hong_qigong"
```

### 问题：对话显示为空

```bash
# 1. 检查对话文件是否存在
ls src/data/dialogues/hong_qigong.zh.json

# 2. 检查 JSON 格式是否正确
npx prettier --check src/data/dialogues/hong_qigong.zh.json

# 3. 检查文件内容
cat src/data/dialogues/hong_qigong.zh.json
```

### 问题：数据库连接失败

```bash
# 1. 检查 .env.local 文件
cat .env.local | grep DATABASE_URL

# 2. 测试数据库连接
psql -U postgres -d jinyong_go

# 3. 重新推送数据库结构
pnpm db:push
```

### 问题：种子数据加载失败

```bash
# 1. 查看详细错误信息
pnpm seed 2>&1 | tee seed-error.log

# 2. 确保数据库表已创建
pnpm db:push

# 3. 检查 JSON 文件格式
npx prettier --check src/data/*.json
```

## 💡 最佳实践

### ✅ 推荐

1. **修改对话**：直接编辑 JSON 文件
2. **测试新功能**：使用 `pnpm db:reset` 重置数据
3. **查看数据**：使用 `pnpm db:studio` 可视化工具
4. **版本控制**：提交 JSON 文件到 Git

### ❌ 避免

1. **不要直接修改生产数据库**（使用后台管理界面）
2. **不要把用户数据放入 JSON**
3. **不要删除 .env.local 中的 DATABASE_URL**
