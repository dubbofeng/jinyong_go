## 🛠️ 技术实现

### Isometric 坐标转换

```typescript
// 网格坐标 → 屏幕坐标
function cartesianToIsometric(gridX: number, gridY: number) {
  const tileWidth = 128;
  const tileHeight = 64;
  const screenX = (gridX - gridY) * (tileWidth / 2);
  const screenY = (gridX + gridY) * (tileHeight / 2);
  return { screenX, screenY };
}

// 屏幕坐标 → 网格坐标
function isometricToCartesian(screenX: number, screenY: number) {
  const tileWidth = 128;
  const tileHeight = 64;
  const gridX = (screenX / (tileWidth / 2) + screenY / (tileHeight / 2)) / 2;
  const gridY = (screenY / (tileHeight / 2) - screenX / (tileWidth / 2)) / 2;
  return { gridX, gridY };
}
```

### Perlin Noise 地形生成

```typescript
import { Noise } from 'noisejs';

function generateTerrain(width: number, height: number, theme: string) {
  const noise = new Noise(Math.random());
  const tiles = [];
  
  for (let y = 0; y < height; y++) {
    for (let x = 0; x < width; x++) {
      // 使用多层噪声创建自然地形
      const value1 = noise.perlin2(x * 0.1, y * 0.1);
      const value2 = noise.perlin2(x * 0.05, y * 0.05) * 0.5;
      const combined = value1 + value2;
      
      const tileType = mapValueToTile(combined, theme);
      tiles.push({ x, y, tile_type: tileType });
    }
  }
  
  return tiles;
}

function mapValueToTile(value: number, theme: string): string {
  // 根据噪声值和主题选择瓦片类型
  if (theme === 'forest') {
    if (value < -0.3) return 'water';
    if (value < 0.2) return 'grass';
    if (value < 0.5) return 'grass_dense';
    return 'forest';
  }
  // ... 其他主题的映射
}
```

### Autotile 系统

Autotile 根据相邻瓦片自动选择正确的瓦片变体：

```typescript
function getAutotileIndex(map: TileMap, x: number, y: number): number {
  const n = hasSameTile(map, x, y - 1);  // 北
  const e = hasSameTile(map, x + 1, y);  // 东
  const s = hasSameTile(map, x, y + 1);  // 南
  const w = hasSameTile(map, x - 1, y);  // 西
  
  // 转换为 4 位二进制索引
  return (n ? 8 : 0) + (e ? 4 : 0) + (s ? 2 : 0) + (w ? 1 : 0);
}
```

## 🎨 AI图片生成系统

### 技术栈
- **AI模型**: Gemini 2.5 Flash Image
- **图片尺寸**: 1024x1024（生成） → 256x256（处理）
- **背景处理**: 白色背景自动转透明

### 生成流程
1. 使用英文提示词生成1024x1024图片
2. 使用 Sharp 调整到256x256
3. 移除白色背景（亮度>240 → 透明）
4. 保存到 `public/generated/map/`

### 提示词格式
```
Single isometric [建筑类型], isolated on pure white background, 
no surrounding landscape. Clean, standalone game asset with complete 
boundaries. Isometric view.

IMPORTANT: Pure white background (#FFFFFF), isolated game asset 
suitable for transparent background removal.
```

// scripts/adjust-map-coordinates.ts
import { db } from '../app/db';
import { maps } from '../src/db/schema';
import { eq } from 'drizzle-orm';

const updates = [
  { mapId: 'huashan_scene', worldX: 20, worldY: 18 },
  { mapId: 'zhongnan_scene', worldX: 22, worldY: 20 },

## 🚀 最佳实践

### 地图设计原则

1. **尺寸选择**
   - 世界地图：64×64（展示全局）
   - 场景地图：32×32（统一尺寸）

2. **主题搭配**
   - 山地主题 → 山门、武馆（华山、武当、昆仑等）
   - 森林主题 → 野外、隐居地（无量山、桃花岛、古墓）
   - 村庄主题 → 城镇、集市（襄阳城、梅庄）
   - 寺庙主题 → 寺院、道观（少林、天龙、道观）
   - 河流主题 → 水乡（海宁）

3. **传送门布局**
   - 世界地图：按章节和地理区域分布
   - 场景地图：在角落放置返回世界地图的传送门
   - 最小间隔5格以上避免拥挤

4. **坐标规划**
   - 使用 worldX 和 worldY 定义在世界地图上的位置
   - 同一章节的地图可以聚集在一个区域
   - 保持整体地图的平衡感

5. **性能优化**
   - 限制物品数量（建筑+装饰+植物 < 100个/地图）
   - 使用 Autotile 减少精灵数量
   - 实施视口裁剪，只渲染可见区域

## 🐛 故障排除

### 常见问题

**Q: 地图生成失败**
- 检查数据库连接（POSTGRES_URL环境变量）
- 确认 items 表中有足够的建筑和装饰物品
- 查看控制台错误信息

**Q: 传送门不显示图标**
- 检查 maps 表的 isometricImage 字段
- 确认图片文件存在于 public/generated/map/
- 运行 batch-process-map-images.ts 处理图片

**Q: 瓦片显示不正确**
- 确认 Autotile 文件存在于 public/game/isometric/autotiles/
- 检查 tileType 值（1=grass, 2=gold, 3=road, 4=boundary, 5=water）
- 清除浏览器缓存

**Q: 传送门不工作**
- 检查 sceneLinkMapId 是否正确
- 确认目标地图存在于数据库
- 查看浏览器控制台的错误信息

**Q: 从小地图返回世界地图后位置不对**
- 检查 worldX 和 worldY 坐标是否设置
- 确认 IsometricGame.tsx 的 confirmPortal 函数逻辑
- 玩家应该出现在该地图在世界地图上的传送门位置

**Q: AI生成的图片有白色背景**
- 运行 batch-process-map-images.ts 移除背景
- 或手动调用 removeBackground 函数
- 检查亮度阈值设置（默认240）
