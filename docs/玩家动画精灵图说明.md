# 玩家行走动画精灵图说明

## 概述

使用Canvas绘制生成的等距视角玩家行走动画精灵图表，确保：
- ✅ 完全透明的背景（RGBA格式）
- ✅ 同一个角色在所有方向保持一致
- ✅ 包含行走中间动作帧
- ✅ 4个方向，每个方向4帧动画

## 文件信息

**路径**: `/public/game/isometric/characters/player_walk_animation.png`

**尺寸**: 512x512 像素（4列 x 4行）

**文件大小**: 14KB

**格式**: PNG RGBA（透明背景）

## 精灵图布局

```
帧数:  0      1      2      3
     +------+------+------+------+
  0  | 向下 | 向下 | 向下 | 向下 |  第1行：向下行走
     | 帧0  | 帧1  | 帧2  | 帧3  |
     +------+------+------+------+
  1  | 向左 | 向左 | 向左 | 向左 |  第2行：向左行走
     | 帧0  | 帧1  | 帧2  | 帧3  |
     +------+------+------+------+
  2  | 向右 | 向右 | 向右 | 向右 |  第3行：向右行走
     | 帧0  | 帧1  | 帧2  | 帧3  |
     +------+------+------+------+
  3  | 向上 | 向上 | 向上 | 向上 |  第4行：向上行走
     | 帧0  | 帧1  | 帧2  | 帧3  |
     +------+------+------+------+
```

## 使用方式

### 1. 在Player类中

```typescript
// PlayerState增加了方向和动画帧字段
interface PlayerState {
  // ...
  direction: 0 | 1 | 2 | 3; // 0=下 1=左 2=右 3=上
  animationFrame: number;   // 当前帧（0-3）
  animationTime: number;    // 动画计时器
}

// 渲染时从精灵图表提取对应帧
const sourceX = this.state.animationFrame * 128;  // 列
const sourceY = this.state.direction * 128;        // 行

ctx.drawImage(
  spriteSheet,
  sourceX, sourceY, 128, 128,  // 源矩形
  renderX, renderY, width, height  // 目标矩形
);
```

### 2. 动画更新逻辑

```typescript
update(deltaTime: number) {
  if (this.state.isMoving) {
    // 根据移动方向更新朝向
    if (Math.abs(dx) > Math.abs(dy)) {
      this.state.direction = dx > 0 ? 2 : 1; // 右或左
    } else {
      this.state.direction = dy > 0 ? 0 : 3; // 下或上
    }
    
    // 更新动画帧
    this.state.animationTime += deltaTime;
    if (this.state.animationTime >= 1/10) {  // 10 FPS
      this.state.animationFrame = (this.state.animationFrame + 1) % 4;
      this.state.animationTime = 0;
    }
  } else {
    // 静止时使用第0帧
    this.state.animationFrame = 0;
  }
}
```

## 角色设计

- **身体**: 蓝色衣服（`#3b82f6`），菱形等距视角
- **头部**: 黄色皮肤（`#fbbf24`），圆形
- **头发**: 黑色（`#1f2937`）
- **武器**: 灰色剑（`#9ca3af`）
- **动画**: 腿部摆动，手臂前后摆动

## 对比AI生成方式

### 为什么使用Canvas而不是AI？

1. **一致性**: AI每次生成的角色都不同，无法保证同一个人
2. **动画连贯性**: AI难以生成连贯的动画序列
3. **透明背景**: AI生成的图片背景往往不是真正透明
4. **可控性**: Canvas绘制可以精确控制每个细节
5. **文件大小**: 14KB vs AI生成的几十KB

### NPC精灵图

NPC仍然使用AI生成（Gemini API），因为：
- NPC不需要动画（静止站立）
- 每个NPC需要独特的外观
- 可以通过后处理移除背景

## 帧率建议

- **推荐**: 8-12 FPS（较慢，适合武侠风格的缓慢行走）
- **当前**: 10 FPS
- **修改**: 在Player类中调整 `ANIMATION_FPS` 常量
