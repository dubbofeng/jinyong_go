# 地图管理功能测试指南

## ✅ 完成功能

### 1. 地图列表页面
**访问路径**: `http://localhost:9999/zh/admin/maps`

**功能特性**:
- ✅ 显示所有地图列表（卡片式布局）
- ✅ 地图统计卡片（总数、世界地图数、场景地图数）
- ✅ 地图类型标签（世界/场景）
- ✅ 地图尺寸信息
- ✅ 空状态提示（无地图时）
- ✅ 快速操作按钮（编辑/查看/删除）

**测试步骤**:
1. 访问 `http://localhost:9999/zh/admin/maps`
2. 应该看到 5 个现有地图（world_map, huashan, shaolin, wudang, taohua）
3. 点击"创建新地图"按钮跳转到生成器

---

### 2. 地图生成器
**访问路径**: `http://localhost:9999/zh/admin/maps/generate`

**功能特性**:
- ✅ 基本信息表单（地图ID、名称、描述）
- ✅ 地图类型选择（世界地图 🌍 / 场景地图 🏯）
- ✅ 地图尺寸选择（32×32 / 48×48 / 64×64）
- ✅ 主题选择（6种主题：森林🌲、山地⛰️、村庄🏘️、河流🌊、沙漠🏜️、寺庙⛩️）
- ✅ 表单验证
- ✅ 生成后自动跳转到编辑器

**测试步骤**:
1. 访问生成器页面
2. 填写表单：
   - 地图ID: `test_village`
   - 名称: `测试村庄`
   - 描述: `新手村场景`
   - 类型: 场景地图
   - 尺寸: 32×32
   - 主题: 村庄
3. 点击"生成地图"
4. 等待生成完成（几秒钟）
5. 自动跳转到地图编辑器

**API 调用**:
```bash
curl -X POST http://localhost:9999/api/maps/generate \
  -H "Content-Type: application/json" \
  -d '{
    "mapId": "test_village",
    "name": "测试村庄",
    "description": "新手村场景",
    "mapType": "scene",
    "width": 32,
    "height": 32,
    "theme": "village"
  }'
```

### 3. 地图编辑器
**访问路径**: `http://localhost:9999/zh/admin/maps/[id]/edit`

**功能特性**:
- ✅ 三栏布局（工具栏/画布/属性面板）

- ✅ 可视化地图画布（32×32 网格）
- ✅ 瓦片点击编辑
- ✅ 属性编辑（地图名称、描述）
- ✅ 实时地图数据显示

**测试步骤**:
1. 从地图列表点击任意地图的"编辑"按钮
2. 或直接访问 `http://localhost:9999/zh/admin/maps/1/edit`
3. 左侧选择工具和瓦片类型
4. 在画布上点击瓦片进行编辑
5. 右侧修改地图属性
6. 点击"保存"按钮保存更改

**当前限制**:
- ⚠️ 瓦片编辑功能已实现UI，但实际保存到数据库的API还需完善
- ⚠️ 物品放置功能UI已就绪，后续需要实现mapItems的CRUD API

## 📊 数据库结构

### maps 表
```sql
id          SERIAL PRIMARY KEY
mapId       VARCHAR(100) UNIQUE    -- 唯一标识符
name        VARCHAR(255)           -- 显示名称
description TEXT                   -- 描述
mapType     VARCHAR(50)            -- world/scene
width       INT                    -- 地图宽度
height      INT                    -- 地图高度
createdAt   TIMESTAMP
updatedAt   TIMESTAMP
```

### mapTiles 表
```sql
id       SERIAL PRIMARY KEY
mapId    INT REFERENCES maps(id)
x        INT                       -- X坐标
y        INT                       -- Y坐标
tileType VARCHAR(50)               -- dirt/gold/wood/fire/water
```

### mapItems 表
```sql
id          SERIAL PRIMARY KEY
mapId       INT REFERENCES maps(id)
itemType    VARCHAR(50)            -- building/npc/portal/decoration
x           INT                    -- X坐标
y           INT                    -- Y坐标
config      JSONB                  -- 物品配置
```

## 🐛 已知问题

1. **地图编辑器参数类型**
   - 需要使用 `use(params)` 来获取异步参数
   - 已在当前版本修复

2. **API mapId 参数**
   - GET /api/maps/[mapId] 使用的是数据库ID（数字）
   - 而不是mapId字段（字符串）
   - 需要根据实际使用场景决定使用哪个
